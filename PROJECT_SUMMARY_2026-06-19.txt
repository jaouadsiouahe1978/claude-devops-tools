✓ Roles                  → Modular, reusable components

✓ Jinja2 Templates      → Dynamic configuration files

✓ Idempotence           → Run N times = same result
✓ Multi-host deployment → 3+ servers at once
✓ Firewall management   → UFW rules via Ansible
✓ Service management    → Systemd via Ansible
✓ Monitoring integration → Prometheus + exporters

DEPLOYMENT OPTIONS
================================================================================

OPTION 1: Docker Compose (Recommended for learning)
- No local VMs needed
- Pre-configured containers (Ubuntu + SSH)
- Mirrors production setup
- Command: docker-compose up -d

OPTION 2: Linux Servers (Production)
- Real Ubuntu/Debian servers
- SSH key-based authentication
- Edit inventory/hosts.ini with IPs
- Command: ansible-playbook playbooks/site.yml -v

OPTION 3: Vagrant (Virtual Machines)
- VirtualBox-based VMs
- Automatic Ansible provisioning
- Command: vagrant up

================================================================================
WORKFLOW
================================================================================

Step 1: Preparation (10-15 min)
├─ Install Ansible
├─ Generate SSH keys
├─ Copy keys to servers
└─ Test connectivity: ansible all -m ping

Step 2: Configuration (5 min)
├─ Edit inventory/hosts.ini
├─ Verify inventory: ansible-inventory --list
└─ Check host connectivity

Step 3: Simulation (5 min)
├─ Dry-run: ansible-playbook --check
├─ Review changes (verbose mode)
└─ Fix any issues

Step 4: Deployment (20-30 min)
├─ Execute: ansible-playbook playbooks/site.yml
├─ Monitor output
└─ Address any errors

Step 5: Validation (10 min)
├─ Test web: curl http://web1.example.com
├─ Test API: curl http://web1.example.com:3000
├─ Test DB: psql -U app_user -d app_db
└─ Test metrics: curl http://web1.example.com:9100/metrics

Step 6: Iteration (5 min per change)
├─ Modify templates/vars
├─ Redeploy with tags
└─ Validate changes

PRACTICAL USE CASES
================================================================================

1. Add a new web server
   - Add to inventory/hosts.ini
   - Run playbook (idempotent)
   - New server configured automatically

2. Update Nginx configuration
   - Modify roles/nginx/templates/nginx.conf.j2
   - Run: ansible-playbook --tags nginx
   - Changes deployed to all web servers

3. Scale application vertically
   - Update group_vars/webservers.yml
   - Redeploy with tags
   - All servers updated consistently

4. Infrastructure documentation
   - Playbooks serve as runbooks
   - Easy to audit what was deployed
   - Version controlled in git

5. Disaster recovery
   - Use same playbooks
   - Rebuild entire stack in minutes
   - Consistent configuration

================================================================================
DEBUGGING COMMANDS
================================================================================

Verbosity levels:
- -v    : Show tasks and results
- -vv   : Show variables and registered values
- -vvv  : Show connection details
- -vvvv : Show debug output

Useful commands:
```bash
# Check syntax
ansible-playbook playbooks/site.yml --syntax-check

# Simulate (dry-run)
ansible-playbook playbooks/site.yml --check

# Run specific role
ansible-playbook playbooks/site.yml --tags nginx

# Run on specific host
ansible-playbook playbooks/site.yml -l web1.example.com

# Ad-hoc commands
ansible webservers -m shell -a "uptime"
ansible dbservers -m apt -a "name=htop" -b

# Debug tasks
ansible-playbook playbooks/site.yml -vvv --step
```
