### 🏗️ Architecture
```
Multi-server web application with:
- Nginx (reverse proxy)
- Node.js + Express (backend)
- PostgreSQL (database)
- Prometheus + Node Exporter (monitoring)
```

### 📦 Ansible Components
1. **6 Reusable Roles**
   - `common/` - System setup, firewall, SSH
   - `nginx/` - Web server and reverse proxy
   - `nodejs/` - Node.js, NVM, PM2, app deployment
   - `postgres/` - Database server setup
   - `prometheus/` - Monitoring backend
   - `monitoring/` - Node Exporter for metrics

2. **Inventory Management**
   - 3 host groups (webservers, dbservers, monitoring)
   - 2 inventory formats (INI and YAML)
   - Support for 2+ webservers, 1 DB, 1 monitoring

3. **Variable Organization**
   - `group_vars/` for group-specific variables
   - `host_vars/` for host-specific overrides
   - Centralized configuration management

4. **Deployment Automation**
   - Main playbook: `playbooks/site.yml`
   - Support for partial deployments with tags
   - Handlers for service restarts
   - Idempotent operations
