================================================================================
WORKFLOWS EXPLIQUÉS
================================================================================

📦 BUILD.YML - Build & Test Pipeline
─────────────────────────────────────
Déclenché par: push sur main/develop, pull_request sur main
Actions:
  1. Checkout code
  2. Setup Docker Buildx
  3. Build image Docker (multi-stage)
  4. Load image locally
  5. Run tests dans le conteneur
  6. Run linting avec pylint
  7. Check image size (< 500MB)
  8. Test endpoints HTTP
  9. Scan avec Trivy
  10. Generate SBOM
  11. Upload artifacts

Matrice: Python 3.10, 3.11, 3.12

🔐 SECURITY-SCAN.YML - Security Scanning
──────────────────────────────────────────
Déclenché par: push, pull_request, schedule (hebdo)
Actions:
  1. Trivy filesystem scan
  2. Secret detection (TruffleHog)
  3. Dependency vulnerability check
  4. Dockerfile linting (Hadolint)
  5. SARIF upload to GitHub Security

🚀 DEPLOY.YML - Deployment to Registry
────────────────────────────────────────
Déclenché par: Tags git (v*) ou workflow_dispatch
Actions:
  1. Checkout with full history
  2. Setup Docker Buildx
  3. Login to Docker Hub
  4. Build and push image
  5. Generate OCI labels
  6. Create deployment summary
  7. Notify Slack (optional)

Environment: staging/production (configurable)

================================================================================
QUICK START
================================================================================

1. Build localement:
   $ docker build -t flask-app:local app/

2. Run avec Docker Compose:
   $ docker-compose up -d
   $ docker-compose exec web curl http://localhost:5000/

3. Tests:
   $ docker run --rm flask-app:local python -m pytest tests/ -v

4. Simuler workflow (avec act):
   $ brew install act
   $ act push -j build

================================================================================
SECRETS À CONFIGURER (GitHub)
================================================================================

Pour vrai déploiement sur Docker Hub:

Settings > Secrets and variables > Actions

Ajouter:
  DOCKER_USERNAME      = votre_username
  DOCKER_PASSWORD      = votre_token_docker
  SLACK_WEBHOOK        = https://hooks.slack.com/... (optionnel)
