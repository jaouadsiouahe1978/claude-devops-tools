🎯 PROJET : GitHub Actions - Pipeline CI/CD Complet

📝 RÉSUMÉ
─────────
Apprenez à créer un pipeline CI/CD complet avec GitHub Actions. Ce projet inclut 
3 workflows professionnels, des tests multi-version Node (16/18/20), un Docker 
multi-stage build, et du security scanning automatique. Vous comprendrez les 
concepts essentiels : jobs parallèles, dépendances, artifacts, secrets, et 
environments avec approbation.

🛠️ TECHNOLOGIES
────────────────
- GitHub Actions (Workflows & Automation)
- Node.js 16/18/20 (Matrix builds)
- Jest (Unit tests with coverage)
- Docker (Multi-stage builds)
- ESLint + Prettier (Code quality)
- Trivy (Security scanning)
- YAML (Configuration as code)

💡 CONCEPTS CLÉ
────────────────
Workflows    → Fichiers YAML = automatisation complète
Jobs         → Unités d'exécution (parallèles ou séquentiels)
Artifacts    → Partage de fichiers entre jobs
Matrix       → Tester sur plusieurs versions simultanément
Secrets      → Variables chiffrées pour les credentials
Environments → Approbations avant déploiement
Cron         → Tâches planifiées (schedule: '0 2 * * 1')
